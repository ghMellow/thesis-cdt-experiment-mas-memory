We evaluate the unguided agentic LLM (no SAST hints) on four network
functions of Free5GC (PCF, UDR, AMF, UDM), each containing at least one
publicly disclosed CVE, using the pipeline described in Sec.~IV. For
each network function the agent is run for 3 independent repetitions;
findings are compared against the official patch by the deterministic,
ground-truth-aware judge (Sec.~IV, \emph{Measurement} phase), producing
the true/false positive and false negative counts that the tables below
are built from. A fifth task with no mapped target CVE (cross-NF) is
reported separately and excluded from the pooled rows, since every one
of its findings is unmatched by construction and would otherwise be
indistinguishable from noise.

Aware of the incompleteness of the ground truth -- limited to publicly
known, mapped CVEs and not covering the full attack surface of the
codebase -- we do not treat Recall, nor the metrics derived from it
(F1), as a measure of completeness: a ``miss'' under this metric may
simply be a vulnerability not yet catalogued, not one the agent failed
to find. The quantitative analysis therefore focuses exclusively on
Precision (and its inverse, Alerts-per-True-Positive), which measures
the false-alarm burden the system imposes on a human analyst,
independently of how complete the reference dataset is. F1 is not
reported, to avoid conveying a false sense of balance between two
properties we are not in a position to both measure.
