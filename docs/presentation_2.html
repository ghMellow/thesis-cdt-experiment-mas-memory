Root Cause

Un Pattern Framework-Specific

Codice vulnerabileif influenceId != "subs-to-notify" { c.String(http.StatusNotFound, "404 page not found") // manca return — esecuzione continua } subscriptionId := c.Param("subscriptionId") s.Processor().HandleDelete(...)
Codice correttoif influenceId != "subs-to-notify" { c.String(http.StatusNotFound, "404 page not found") return // esecuzione si ferma } subscriptionId := c.Param("subscriptionId") s.Processor().HandleDelete(...)
  • Gin non ferma Go dopo c.String() — la funzione continua, il processor viene chiamato su un path rifiutato
  • Non è OWASP top 10, non assomiglia a SQL injection o XSS — non è nei dataset tipici di security review
  • Richiede 3 livelli simultanei: semantica Go + comportamento Gin + flow control 5G SBI
